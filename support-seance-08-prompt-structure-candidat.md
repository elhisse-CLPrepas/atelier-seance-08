# Support séance 08 — Prompt de structuration candidat

## Objectif du support

Ce support aide chaque candidat à préparer, structurer et améliorer son livrable pour la séance 08.

Il sert à transformer le travail amorcé en séance 07 en une version claire, présentable et exploitable, en utilisant l’IA comme assistant de structuration, de reformulation et de vérification.

## Contenu de la séance 08

La séance 08 est consacrée à la finalisation, au test et à l’amélioration des productions réalisées à partir de la séance 07.

Pendant cette séance, le candidat doit être capable de :

- présenter rapidement son projet ou son livrable ;
- expliquer le besoin traité et le public visé ;
- montrer ce qui a été produit depuis la séance 07 ;
- identifier les points forts et les points faibles de sa production ;
- utiliser les retours reçus pour améliorer le livrable ;
- produire une version plus claire, plus fiable et plus présentable.

## Déroulé candidat proposé

| Étape | Action candidat | Résultat attendu |
| --- | --- | --- |
| 1 | Présenter le brouillon issu de la séance 07 | Le point de départ est clair |
| 2 | Relire et nettoyer le contenu | Les doublons et formulations faibles sont supprimés |
| 3 | Compléter les parties manquantes | Le livrable devient plus complet |
| 4 | Reformuler avec précision | Le vocabulaire est adapté au public cible |
| 5 | Tester ou faire relire | Les incompréhensions sont repérées |
| 6 | Finaliser une version présentable | Le livrable peut être partagé ou démontré |
| 7 | Préparer une mini-présentation | Le candidat sait expliquer son travail simplement |

## Prompt principal à copier-coller

Copie-colle le prompt ci-dessous dans ton outil d’IA, puis remplace les éléments entre crochets par tes informations.

```text
Tu es mon assistant pédagogique pour préparer la séance 08.

Contexte :
Je suis candidat dans un programme de montée en compétences IA.
En séance 07, j’ai commencé à produire un livrable à partir d’un besoin concret.
En séance 08, je dois présenter une version plus claire, plus structurée et plus utile.

Voici mon projet ou mon livrable :
[TITRE OU NOM DU PROJET]

Public cible :
[À QUI S’ADRESSE LE LIVRABLE ?]

Besoin traité :
[QUEL PROBLÈME OU OBJECTIF LE LIVRABLE DOIT-IL TRAITER ?]

Version actuelle de mon travail :
[COLLER ICI LE BROUILLON, LE PLAN, LE TEXTE, LA FICHE, LE SCRIPT OU LA DESCRIPTION DU PROTOTYPE]

Ce que je veux obtenir pour la séance 08 :
- une structure plus claire ;
- des formulations plus professionnelles ;
- une liste des parties à compléter ;
- des questions de feedback à poser ;
- une mini-présentation de 3 à 5 minutes.

Ta mission :
1. Analyse mon travail actuel.
2. Repère les points forts.
3. Repère les éléments flous, incomplets ou inutiles.
4. Propose une structure améliorée.
5. Reformule les passages trop génériques.
6. Liste les informations à vérifier.
7. Propose 3 questions de feedback à poser en séance 08.
8. Prépare un plan de présentation orale en 5 minutes maximum.

Contraintes :
- Ne rajoute pas de fausses informations.
- Signale clairement ce qui doit être vérifié.
- Garde un style simple, clair et adapté à mon public cible.
- Explique les améliorations proposées.
```

## Prompt court de nettoyage

À utiliser si le livrable est déjà avancé et doit surtout être clarifié.

```text
Relis le contenu suivant et aide-moi à le nettoyer pour la séance 08.

Objectif : obtenir une version claire, structurée, sans doublons et compréhensible par une personne extérieure au projet.

Contenu à améliorer :
[COLLER LE CONTENU]

Je veux que tu me proposes :
1. les doublons à supprimer ;
2. les formulations trop vagues ;
3. une version réécrite plus claire ;
4. les informations manquantes ;
5. les points à vérifier avant présentation.
```

## Prompt pour préparer la présentation orale

```text
Aide-moi à préparer une présentation orale de 3 à 5 minutes pour la séance 08.

Voici mon livrable :
[COLLER LE RÉSUMÉ OU LE CONTENU DU LIVRABLE]

Prépare :
1. une phrase d’introduction ;
2. le besoin traité ;
3. le public cible ;
4. ce que j’ai produit ;
5. ce que l’IA m’a aidé à faire ;
6. ce que j’ai corrigé ou amélioré moi-même ;
7. les limites actuelles ;
8. les prochaines améliorations possibles ;
9. une conclusion courte.

Le style doit être simple, naturel et professionnel.
```

## Prompt pour demander du feedback

```text
À partir de mon livrable, propose-moi 5 questions de feedback à poser aux autres candidats ou au formateur pendant la séance 08.

Mon livrable :
[COLLER LE CONTENU OU LE RÉSUMÉ]

Les questions doivent m’aider à vérifier :
- la clarté ;
- l’utilité ;
- la cohérence ;
- l’adaptation au public cible ;
- les améliorations prioritaires.
```

## Checklist candidat avant la séance 08

Avant d’arriver en séance 08, je vérifie que :

- [ ] mon livrable a un titre clair ;
- [ ] le besoin traité est compréhensible ;
- [ ] le public cible est identifié ;
- [ ] les doublons ont été supprimés ;
- [ ] les formulations vagues ont été reformulées ;
- [ ] les informations importantes ont été vérifiées ;
- [ ] les limites ou incertitudes sont indiquées ;
- [ ] je peux présenter mon travail en moins de 5 minutes ;
- [ ] j’ai préparé au moins 2 questions de feedback ;
- [ ] je sais expliquer ce que l’IA m’a apporté et ce que j’ai retravaillé moi-même.

## Livrable attendu en fin de séance 08

À la fin de la séance 08, le candidat doit disposer d’une version améliorée de son livrable comprenant :

- une structure claire ;
- un contenu relu et nettoyé ;
- des informations vérifiées ou signalées comme à vérifier ;
- une présentation courte prête à être utilisée ;
- une liste d’améliorations prioritaires pour la suite.
